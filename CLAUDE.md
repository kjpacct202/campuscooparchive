# CLAUDE.md — project brief for the build session

This file orients a Claude Code session working in this repo. Read it first.

## What this project is

The **Campus COOP Archive** (`campuscooparchive`) is a fully-sourced catalog +
analysis layer of US higher-ed **continuity** plans (COOP / BCP / academic
continuity / IT-DR). It is the continuity sibling of the **Higher Ed EOP Atlas**
(live at `campus-eop-archive.vercel.app`) and the **Campus Alert Archive**
(`campusalertarchive.com`). The data foundation is built; the website and
deployment are not (see `HANDOFF.md`).

GitHub remote (not yet initialized): `git@github.com:kjpacct202/campuscooparchive.git`

## The one rule that governs everything

**Every datapoint must be traceable to a primary source a skeptic can check.**
Never invent a URL or a quote. Every plan record has `source_url`,
`landing_page_url`, a verbatim `evidence_quote`, a `verification` badge
(`opened-and-confirmed` | `link-only`), and `date_accessed`. If you add or edit
records, preserve this contract. Source documents are **linked, not
redistributed**.

## Data contract (don't break these)

- **`data/plans.json`** — array, one object per plan. Authoritative shape:
  `schema/plan.schema.json`; fields: `schema/field_dictionary.md`; vocab:
  `schema/taxonomies.md`. Join key `plan_id` (kebab slug). `institution_id` is
  derived as `slugify(institution_name)`.
- **`data/institutions.json`**, **`data/stats.json`**, **`data/plans.csv`**,
  **`data/institutions.csv`** are **generated** by `scripts/build_dataset.py`.
  Do not hand-edit them; edit `data/raw/*.json` and rebuild.
- **`data/deep_analysis.json`** — generated by `scripts/merge_deep.py` from
  `data/deep/batch_*.json`. Deep records score against the 22-component
  benchmark; `benchmark_present = 22 − len(benchmark_missing)`. The 22 valid keys
  are listed in `docs/CONTINUITY_BENCHMARK.md`, `scripts/merge_deep.py`
  (`BENCHMARK_KEYS`), and `docs/CODING_GUIDE.md` — keep all three in sync if you
  ever change them.
- **`data/raw/*.json`** — immutable provenance (one file per discovery lane).
  Add new lanes as new files; the build picks up any `*.json` array.

## Pipeline (pure Python 3 stdlib, no deps, idempotent)

```bash
python3 scripts/build_dataset.py     # raw -> plans/institutions/stats
python3 scripts/merge_deep.py        # deep batches -> deep_analysis.json
python3 scripts/build_dataset.py     # re-run: folds deep scores into plans/stats
python3 scripts/build_dashboard.py   # -> site/index.html (offline prototype)
python3 scripts/check_links.py --dry # URL inventory; drop --dry for a live check
```

Running any script twice with no input change produces identical output. The
offline `site/index.html` embeds the data inline (no network) and is the
reference for the production UI's behavior.

## Conventions

- **Python:** standard library only. No pip installs. Keep scripts idempotent and
  side-effect-free except for their named outputs.
- **Slugs:** kebab-case, ASCII. Multiple plans for one institution get a suffix
  (`-bcp`, `-itdr`, `-academic`); `build_dataset.py` resolves `plan_id`
  collisions automatically.
- **Enums:** closed lists in `schema/taxonomies.md`. Add a value there before
  using it.
- **Discovery flags are conservative lower bounds.** The deep layer supersedes
  them with full-text-verified values + quotes.

## What NOT to do

- Don't hand-edit generated files (`plans.json/csv`, `institutions.*`,
  `stats.json`, `deep_analysis.json`, `site/index.html`) — regenerate them.
- Don't loosen the verifiability standard or fabricate any field.
- Don't redistribute source PDFs; link to them.
